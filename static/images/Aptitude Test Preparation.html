## ✅ Section 4: **Power BI – Answers**

1. **Filters**: Visual-level, Page-level, Report-level, Drillthrough, Top N

2. **DirectQuery** pulls data in real-time; **Import** pulls data into Power BI’s cache

3. **DAX** (Data Analysis Expressions) – e.g., `Total Sales = SUM(Sales[Amount])`

4. Go to Data View → New column → Write DAX expression

5. **Calculated column** stores value in model; **Measure** calculates on the fly (e.g., aggregates)

6. Relationships define how tables connect (1-to-many, many-to-one, etc.)

7. Use **bridge tables** or composite models

8. **Slicers** filter visuals dynamically (e.g., by date, region)

9. **Drill-down** allows navigating from summary to detail level within a chart

10. Through **Power BI Service** → Scheduled Refresh settings

11. `ALL()` removes filters in a DAX expression

12. Bookmarks capture visual/page states for storytelling or navigation

13. Optimize using:
- Avoid too many visuals
- Use star schema
- Disable auto-date/time
- Reduce cardinality

14. **Architecture**: Power BI Desktop → Power BI Service → Data Gateway → Power BI Mobile

15. Create roles in Modeling tab → Use `DAX filters` → Apply in Power BI Service
